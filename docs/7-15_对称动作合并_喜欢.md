# 7/15 对称动作合并_喜欢

请在现有 MicroMove MVP 基础上完成以下更新。

不要重构整体流程，不要删除或合并底层动作数据，不要修改现有动作图片、动作说明和安全提示内容。首页推荐跟练中现有的左右动作绑定、对应图片展示、完成记录等逻辑需要继续保留。

---

## 一、动作库中的对称动作合并展示

### 1. 合并范围

将目前动作库中成对出现的动作，在展示层合并为一个动作组，包括：

- 左右对称动作；
- 前后或两个方向成对完成的动作；
- 目前首页推荐方案中已经被识别为需要连续完成的配对动作。

例如：

- “左侧颈部侧弯”与“右侧颈部侧弯”，在动作库中合并显示为“颈部侧屈拉伸”。

### 2. 只合并前端展示，不合并底层数据

- 左右两侧仍保留各自原有的 `exerciseId`。
- 不删除、不合并、不重命名底层动作记录。
- 每一侧继续使用自己对应的动作图片、动作说明和完成记录。
- 可新增独立的展示层配对配置，例如 `pairedExerciseGroups`，将一个动作组与多个原始 `exerciseId` 关联。
- 首页现有的配对推荐和连续跟练逻辑保持不变。

### 3. 动作库卡片展示

每组对称动作在动作库中只显示一张卡片：

- 标题使用不区分方向的组合名称，例如“颈部侧屈拉伸”；
- 封面图选择其中一个方向即可，默认可使用左侧动作图片；
- 卡片中的部位、主要肌肉、时长、姿势、空间及强度等信息，沿用该动作组已有内容；
- 不再分别显示“左侧”和“右侧”两张卡片。

---

## 二、重新区分“动作库浏览”和“首页快速跟练”

### 1. 动作库的定位

动作库主要用于：

- 浏览有哪些动作；
- 查看动作怎么做；
- 阅读动作指引和安全提示；
- 用户主动选择某个动作进行跟练。

因此，从动作库点击动作后，不得立刻进入倒计时。

### 2. 动作库详情页

用户点击动作库卡片后，进入完整动作详情页，页面应展示：

- 完整动作示意图；
- 动作名称；
- 主要训练或拉伸部位；
- 动作指引；
- 安全提示；
- “喜欢”按钮；
- 页面底部主按钮：**立即跟练**。

对于合并后的对称动作组：

- 详情页只需要以其中一侧作为动作学习示例，默认使用左侧；
- 图片、动作指引和安全提示均以该侧为例；
- 页面中应明确该动作跟练时会完成两侧，例如显示“跟练时将依次完成左右两侧”。

### 3. 动作库路径取消10秒学习阶段

动作库路径：

> 动作库卡片 → 动作详情页 → 点击“立即跟练” → 直接进入30秒跟练

要求：

- 不再进入首次动作的10秒学习倒计时；
- 动作详情页本身承担动作学习和确认功能；
- 不受 `learnedExerciseIds` 是否已经记录影响。

首页推荐路径保持原有机制：

- 用户从首页入口进入推荐方案时，首次动作仍按现有规则进入10秒学习确认；
- 已学习动作仍可直接进入正式跟练；
- 不修改首页路径的 `learnedExerciseIds` 逻辑。

---

## 三、从动作库跟练对称动作时自动完成另一侧

当用户在对称动作组详情页点击“立即跟练”后：

1. 先进入详情页中展示的方向，默认左侧；
2. 显示该侧对应的图片并进行30秒倒计时；
3. 当该侧倒计时结束，或用户点击“跳过”后；
4. 自动进入3秒过渡倒计时；
5. 3秒结束后自动进入对称的另一侧；
6. 第二侧必须显示自己对应的图片、动作名称和方向；
7. 第二侧继续按照现有30秒跟练逻辑运行。

### 3秒过渡页

过渡页需要明确提示下一侧，例如：

- “准备换到右侧”
- 大号数字显示“3、2、1”

页面下方提供次级文字按钮：

- **取消另一侧**

点击后：

- 不再进入另一侧；
- 结束本次从动作库发起的跟练；
- 完成记录沿用现有规则，只记录实际完成的动作，不新增重复记录。

### 非对称动作

普通单方向动作点击“立即跟练”后，只完成该动作一次，不进入3秒换侧流程。

## 四、新增“喜欢”功能

产品中统一使用“喜欢”，不要使用“收藏”或“收藏夹”。

当前版本不需要增加单独的“我喜欢的动作”页面，但喜欢状态需要影响动作库排序和首页推荐。

### 1. 动作库卡片上的爱心

将动作库卡片右上角现有的箭头替换为爱心按钮：

- 未喜欢：空心爱心；
- 已喜欢：红色实心爱心；
- 点击爱心只切换喜欢状态，不进入详情页；
- 点击卡片的其他区域才进入动作详情页。

### 2. 其他入口中的喜欢按钮

用户也可以在以下位置切换喜欢状态：

- 动作库详情页；
- 从首页进入的首次学习页；
- 正式跟练倒计时页；
- 动作完成后的信息页，如当前页面结构适合放置。

各位置必须读取并修改同一份喜欢状态，不能各自独立。

### 3. 对称动作组的喜欢状态

对称动作按整个动作组处理：

- 用户喜欢“颈部侧屈拉伸”，即视为喜欢整个左右动作组；
- 不分别设置“喜欢左侧”和“喜欢右侧”；
- 无论用户在左侧还是右侧的跟练页面点击喜欢，动作组爱心状态都同步变化；
- 动作库中该组合卡片也同步显示相同状态。

### 4. 本地保存

喜欢状态使用本地存储保存，例如：

- `likedExerciseGroupIds`
- `likedExerciseIds`
- `likedAt`

需要记录喜欢时间，以支持“最近喜欢优先”。

取消喜欢后，应同时移除对应的喜欢时间记录。

## 五、动作库中的喜欢排序

在动作库的“全部”和各身体部位分类中，排序规则为：

1. 已喜欢动作优先；
2. 多个已喜欢动作之间，最近喜欢的排在最前面；
3. 未喜欢动作维持原有顺序；
4. 取消喜欢后，该动作返回未喜欢区域，并恢复到原有相对顺序。

对称动作组作为一个整体参与排序。

切换“全部、肩颈、上背、胸肩、腰背”等类别时，都需要实时按照相同规则排序。

## 六、首页推荐方案增加多样性

### 1. 当前问题

用户从首页进入推荐跟练时，多次打开后第一个动作经常相同，导致用户感觉每次方案都一样，不愿继续使用推荐入口。

### 2. 推荐目标

在满足用户已选择的以下条件后：

- 身体部位；
- 坐姿或站姿；
- 空间和其他现有筛选条件；
- 对称动作绑定规则；

尽可能生成更多样的动作组合。

### 3. 具体规则

- 候选动作筛选完成后，不要始终按照动作库固定顺序取第一个动作；
- 每次生成方案时，对符合条件的候选动作进行随机或加权打乱；
- 在候选动作足够的情况下，本次方案的第一个动作不得与上一次方案的第一个动作相同；
- 同一套方案中尽量避免重复动作；
- 尽量避免连续两次生成完全相同的动作组合；
- 左右或前后对称动作仍作为绑定组合处理，不能只随机抽中其中一侧；
- 每个动作正式跟练时，继续显示该方向对应的正确图片。

可在本地记录：

- `lastSessionFirstExerciseId`
- `lastSessionExerciseIds`

用于避免连续重复。

---

## 七、喜欢状态对首页推荐的影响

用户喜欢的动作，在首页推荐方案中应获得更高的推荐优先级，但不能变成固定推荐。

要求：

- 喜欢动作提高被选中的概率；
- 不保证每一套方案都一定出现喜欢动作；
- 不得因为用户喜欢某个动作，就使其每次都成为第一个动作；
- 仍需遵守“本次第一个动作尽量不与上次相同”的规则；
- 仍需保留新动作和其他合适动作的曝光；
- 对称动作组喜欢后，推荐时仍按完整动作组加入方案。

建议使用加权随机，而不是直接将喜欢动作固定排在最前面。

## 八、记录与兼容要求

### 1. 完成记录

- 从动作库点击“立即跟练”完成的动作，继续计入现有历史记录和数据导出；
- 对称动作的左右两侧分别按照实际完成情况记录；
- 不得重复记录同一侧；
- 点击“跳过”或取消另一侧时，沿用现有跳过和完成判定规则。

### 2. 保持现有功能

以下功能不得受到本次修改影响：

- 首页入口生成推荐方案；
- 首页中的左右动作绑定；
- 首次动作10秒学习确认；
- `learnedExerciseIds`；
- 暂停、跳过、重来、上一个；
- 动作完成记录；
- 历史统计；
- CSV/JSON 导出；
- 本地存储中的已有用户数据。

### 3. 数据兼容

- 不清空用户已有本地数据；
- 不改变已有 `exerciseId`；
- 不因展示层合并导致历史记录无法读取；
- 新增字段时提供默认值，避免旧用户页面报错。

## 九、验收标准

完成后请逐项检查：

### 对称动作展示

- [ ] 左右对称动作在动作库中只显示一张组合卡片；
- [ ] 底层左右动作仍保持独立；
- [ ] 卡片封面只展示一侧，不影响跟练时切换正确图片；
- [ ] 普通非对称动作不受影响。

### 动作库详情与跟练

- [ ] 点击动作库卡片后先进入完整详情页；
- [ ] 不会一点击卡片就立刻开始倒计时；
- [ ] 详情页显示完整图片、动作指引和安全提示；
- [ ] 页面底部有“立即跟练”；
- [ ] 动作库路径不再出现10秒学习倒计时；
- [ ] 首页推荐路径仍保留原有10秒学习逻辑。

### 对称动作连续跟练

- [ ] 第一侧结束或跳过后出现3秒换侧倒计时；
- [ ] 3秒后自动进入另一侧；
- [ ] 另一侧显示对应的正确图片和方向；
- [ ] 过渡页可点击“取消另一侧”；
- [ ] 普通动作不会错误进入换侧流程。

### 喜欢功能

- [ ] 动作库卡片右上角箭头已替换为空心/实心爱心；
- [ ] 点击爱心不会进入详情页；
- [ ] 喜欢状态可在动作库详情页和跟练流程中同步修改；
- [ ] 对称动作组只维护一个喜欢状态；
- [ ] 刷新页面后喜欢状态仍然保留；
- [ ] 最近喜欢的动作排在更前面；
- [ ] 取消喜欢后排序恢复正常。

### 首页推荐多样性

- [ ] 连续进入首页推荐时，第一个动作不再固定；
- [ ] 候选动作足够时，连续两次第一个动作不同；
- [ ] 喜欢动作被优先考虑，但不会每次固定出现；
- [ ] 对称动作仍完整绑定；
- [ ] 不影响现有筛选条件和完成记录。
